Βασικές Αρχές PHP

1. Εισαγωγή στην PHP

Η PHP (PHP: Hypertext Preprocessor) είναι μια ευρέως χρησιμοποιούμενη γλώσσα προγραμματισμού ανοιχτού κώδικα, ειδικά σχεδιασμένη για την ανάπτυξη διαδικτυακών εφαρμογών. Εκτελείται στην πλευρά του διακομιστή (server-side) και μπορεί να ενσωματωθεί στην HTML, επιτρέποντας τη δημιουργία δυναμικού περιεχομένου.

Σε αντίθεση με τη JavaScript, που εκτελείται στον περιηγητή του χρήστη, η PHP εκτελείται στον διακομιστή και παράγει HTML που αποστέλλεται στον περιηγητή. Αυτό σημαίνει ότι ο χρήστης δε βλέπει τον κώδικα PHP, αλλά μόνο το αποτέλεσμά του.

Βασικά Σημεία:

  • Η PHP είναι μια γλώσσα προγραμματισμού που εκτελείται στην πλευρά του διακομιστή (server-side)
  • Είναι ιδανική για την ανάπτυξη δυναμικών ιστοσελίδων και διαδικτυακών εφαρμογών
  • Μπορεί εύκολα να ενσωματωθεί στην HTML
  • Είναι ανοιχτού κώδικα και διατίθεται δωρεάν
  • Υποστηρίζεται από την πλειονότητα των διακομιστών ιστού και λειτουργικών συστημάτων
  • Συνεργάζεται άψογα με πολλές βάσεις δεδομένων, όπως MySQL, PostgreSQL, Oracle, κ.ά.

2. Εγκατάσταση Περιβάλλοντος

Για να ξεκινήσετε με την PHP, χρειάζεστε ένα περιβάλλον ανάπτυξης που περιλαμβάνει έναν διακομιστή ιστού, την PHP και, συνήθως, ένα σύστημα διαχείρισης βάσεων δεδομένων. Υπάρχουν διάφοροι τρόποι να εγκαταστήσετε αυτό το περιβάλλον, ανάλογα με το λειτουργικό σας σύστημα και τις ανάγκες σας.

Πακέτα AMP (Apache, MySQL, PHP)

Τα πακέτα AMP είναι προ-διαμορφωμένα πακέτα που περιλαμβάνουν όλα τα απαραίτητα συστατικά:

  • XAMPP: Διασταυρούμενη πλατφόρμα (Windows, Linux, macOS)
  • WAMP: Για Windows
  • MAMP: Για macOS
  • LAMP: Για Linux

Εγκατάσταση του XAMPP

  1. Επισκεφθείτε την επίσημη ιστοσελίδα του XAMPP
  2. Κατεβάστε την έκδοση που ταιριάζει στο λειτουργικό σας σύστημα
  3. Εκτελέστε το αρχείο εγκατάστασης και ακολουθήστε τις οδηγίες του οδηγού εγκατάστασης
  4. Μετά την εγκατάσταση, εκκινήστε τον πίνακα ελέγχου του XAMPP
  5. Ξεκινήστε τις υπηρεσίες Apache και MySQL
  6. Τα αρχεία PHP σας θα πρέπει να τοποθετηθούν στον φάκελο htdocs (συνήθως C:\xampp\htdocs\ σε Windows)

Σημείωση: Για λόγους ασφάλειας, το XAMPP είναι κατάλληλο για περιβάλλον ανάπτυξης, αλλά όχι για περιβάλλον παραγωγής (production).

Έλεγχος Εγκατάστασης

Για να ελέγξετε αν η PHP έχει εγκατασταθεί και λειτουργεί σωστά:

  1. Δημιουργήστε ένα νέο αρχείο κειμένου και ονομάστε το info.php
  2. Προσθέστε τον ακόλουθο κώδικα:
<?php
phpinfo();
?>
  1. Αποθηκεύστε το αρχείο στον φάκελο htdocs του διακομιστή σας
  2. Ανοίξτε έναν περιηγητή και πληκτρολογήστε http://localhost/info.php
  3. Θα δείτε μια σελίδα με λεπτομερείς πληροφορίες για την εγκατάσταση της PHP σας

Προσοχή: Για λόγους ασφαλείας, μην αφήνετε το αρχείο phpinfo() προσβάσιμο σε περιβάλλοντα παραγωγής, καθώς αποκαλύπτει πληροφορίες για τη διαμόρφωση του συστήματός σας.

Ολοκληρωμένα Περιβάλλοντα Ανάπτυξης (IDE)

Για πιο αποτελεσματική ανάπτυξη, εξετάστε τη χρήση ενός IDE ή ενός προηγμένου επεξεργαστή κώδικα:

  • Notepad++: Απλός αλλά αποτελεσματικός επεξεργαστής κώδικα για Windows

3. Σύνταξη και Δομή

Βασική Σύνταξη της PHP

Ο κώδικας PHP περικλείεται σε ειδικές ετικέτες έναρξης και λήξης που επιτρέπουν την εναλλαγή μεταξύ PHP και HTML:

<!DOCTYPE html>
<html>
<head>
    <title>Η πρώτη μου σελίδα PHP</title>
</head>
<body>
    <h1>Καλώς ήρθατε</h1>
    
    <?php
    // Αυτό είναι κώδικας PHP
    echo "<p>Σήμερα είναι " . date("d/m/Y") . "</p>";
    ?>
    
    <p>Αυτό είναι απλή HTML.</p>
    
    <?php
    // Περισσότερος κώδικας PHP
    echo "<p>Η τρέχουσα ώρα είναι " . date("H:i:s") . "</p>";
    ?>
</body>
</html>

Επεξήγηση:

  • Ο κώδικας PHP ξεκινά με <?php και τελειώνει με ?>
  • Μπορείτε να εναλλάσσετε μεταξύ PHP και HTML όσες φορές χρειάζεται μέσα στο ίδιο αρχείο
  • Κάθε εντολή PHP πρέπει να τελειώνει με ερωτηματικό (;)
  • Τα σχόλια δημιουργούνται με // για μια γραμμή ή /* */ για πολλαπλές γραμμές
  • Η συνάρτηση echo χρησιμοποιείται για την εμφάνιση περιεχομένου στην ιστοσελίδα

Συμβουλή: Στις σύγχρονες εφαρμογές PHP, συχνά χρησιμοποιείται μόνο η ετικέτα έναρξης <?php χωρίς ετικέτα λήξης ?> στο τέλος του αρχείου, ειδικά σε αρχεία που περιέχουν μόνο PHP. Αυτό βοηθά στην αποφυγή ακούσιων κενών διαστημάτων που μπορεί να προκαλέσουν προβλήματα στο περιεχόμενο της ιστοσελίδας.

4. Μεταβλητές και Τύποι Δεδομένων

Δήλωση Μεταβλητών

Στην PHP, οι μεταβλητές ξεκινούν πάντα με το σύμβολο του δολαρίου ($) και ακολουθείται από το όνομα της μεταβλητής. Τα ονόματα μεταβλητών είναι ευαίσθητα στα κεφαλαία/πεζά:

<?php
$name = "Γιάννης";
$age = 25;
$isStudent = true;

// Εμφάνιση μεταβλητών
echo $name;                // Εμφανίζει: Γιάννης
echo "<br>";
echo $age;                 // Εμφανίζει: 25
echo "<br>";
echo $isStudent;           // Εμφανίζει: 1 (η boolean true εμφανίζεται ως 1)
echo "<br>";

// Συνδυασμός κειμένου και μεταβλητών
echo "Ονομάζομαι $name και είμαι $age ετών.";
// Εμφανίζει: Ονομάζομαι Γιάννης και είμαι 25 ετών.
?>

Κανόνες Ονοματοδοσίας Μεταβλητών:

  • Πρέπει να ξεκινούν με γράμμα ή κάτω παύλα (_)
  • Μπορούν να περιέχουν μόνο αλφαριθμητικούς χαρακτήρες και κάτω παύλες (A-z, 0-9, και _)
  • Δεν μπορούν να περιέχουν κενά ή ειδικούς χαρακτήρες όπως !, @, #, %, κλπ.
  • Είναι ευαίσθητες στα κεφαλαία/πεζά, οπότε οι $name, $Name και $NAME είναι τρεις διαφορετικές μεταβλητές

Δυναμική Τυποποίηση

Η PHP είναι μια γλώσσα με δυναμική τυποποίηση, που σημαίνει ότι δεν χρειάζεται να δηλώσετε τον τύπο μιας μεταβλητής όταν τη δημιουργείτε. Ο τύπος μιας μεταβλητής καθορίζεται από την τιμή που της αποδίδεται και μπορεί να αλλάξει κατά τη διάρκεια της εκτέλεσης:

<?php
$var = "Γεια σου";     // Η $var είναι συμβολοσειρά (string)
echo $var . "<br>";

$var = 5;              // Τώρα η $var είναι ακέραιος (integer)
echo $var . "<br>";

$var = 3.14;           // Τώρα η $var είναι αριθμός κινητής υποδιαστολής (float)
echo $var . "<br>";

$var = true;           // Τώρα η $var είναι boolean
echo $var . "<br>";
?>

Σημείωση: Παρόλο που η δυναμική τυποποίηση προσφέρει ευελιξία, είναι καλή πρακτική να διατηρείτε συνεπή τον τύπο μιας μεταβλητής σε όλο τον κώδικά σας για την αποφυγή απροσδόκητων σφαλμάτων και για καλύτερη αναγνωσιμότητα.

Βασικοί Τύποι Δεδομένων

Η PHP υποστηρίζει διάφορους τύπους δεδομένων:

Τύπος Περιγραφή Παράδειγμα
String Αλφαριθμητικό (κείμενο) $name = "Μαρία";
Integer Ακέραιος αριθμός $age = 25;
Float/Double Αριθμός κινητής υποδιαστολής $price = 19.99;
Boolean Λογική τιμή (true/false) $isActive = true;
Array Πίνακας (συλλογή τιμών) $colors = array("κόκκινο", "πράσινο", "μπλε");
Object Αντικείμενο (στιγμιότυπο μιας κλάσης) $person = new Person();
NULL Ειδικός τύπος που αντιπροσωπεύει μεταβλητή χωρίς τιμή $result = NULL;
Resource Αναφορά σε εξωτερικό πόρο (π.χ. σύνδεση βάσης δεδομένων) $file = fopen("test.txt", "r");

Έλεγχος Τύπου

Υπάρχουν διάφορες συναρτήσεις για τον έλεγχο του τύπου μιας μεταβλητής:

<?php
$var = "Γεια σου";

// Ελέγχοντας τον τύπο με συναρτήσεις
var_dump($var);       // Εμφανίζει λεπτομερείς πληροφορίες για τη μεταβλητή
echo "<br>";

echo gettype($var);   // Επιστρέφει τον τύπο ως συμβολοσειρά
echo "<br>";

// Έλεγχος αν μια μεταβλητή είναι συγκεκριμένου τύπου
echo is_string($var);    // Επιστρέφει true (εμφανίζεται ως 1)
echo "<br>";
echo is_int($var);       // Επιστρέφει false (δεν εμφανίζεται τίποτα)
echo "<br>";
echo is_bool($var);      // Επιστρέφει false
echo "<br>";
echo is_array($var);     // Επιστρέφει false
echo "<br>";
echo is_null($var);      // Επιστρέφει false
?>

Μετατροπή Τύπων (Type Casting)

Μπορείτε να μετατρέψετε μια μεταβλητή από έναν τύπο σε άλλον με δύο τρόπους:

<?php
// Χρησιμοποιώντας συναρτήσεις μετατροπής
$str = "123";
$num = intval($str);    // Μετατροπή σε ακέραιο
echo $num . "<br>";  // Εμφανίζει: 123

// Χρησιμοποιώντας την τεχνική του casting
$str = "456";
$num = (int)$str;       // Μετατροπή σε ακέραιο
echo $num . "<br>";  // Εμφανίζει: 456

// Παραδείγματα διαφόρων μετατροπών
$float = 3.14;
$int = (int)$float;     // Μετατροπή σε ακέραιο
echo $int . "<br>";  // Εμφανίζει: 3 (το δεκαδικό μέρος αποκόπτεται)

$bool = true;
$str = (string)$bool;   // Μετατροπή σε συμβολοσειρά
echo $str . "<br>";  // Εμφανίζει: 1

$str = "0";
$bool = (bool)$str;     // Μετατροπή σε boolean
echo $bool . "<br>";  // Εμφανίζει: (τίποτα, γιατί το false δεν εμφανίζει τίποτα)
?>

Προσοχή: Κατά τη μετατροπή τύπων, είναι σημαντικό να κατανοήσετε πώς η PHP χειρίζεται τις διάφορες περιπτώσεις. Για παράδειγμα, κατά τη μετατροπή από string σε integer, η PHP θα προσπαθήσει να εξαγάγει έναν αριθμό από την αρχή της συμβολοσειράς, αλλά θα σταματήσει μόλις συναντήσει μη αριθμητικό χαρακτήρα.

Σταθερές

Οι σταθερές είναι αναγνωριστικά (ονόματα) για απλές τιμές που δεν μπορούν να αλλάξουν κατά τη διάρκεια της εκτέλεσης του σεναρίου. Οι σταθερές είναι καθολικές διαθέσιμες σε όλο το σενάριο.

<?php
// Ορισμός σταθερών
define("GREETING", "Καλωσήρθατε στην ιστοσελίδα μας!");
define("PI", 3.14159);
define("SITE_URL", "http://www.example.com/");

// Χρήση σταθερών
echo GREETING . "<br>";
echo "Η τιμή του π είναι περίπου " . PI . "<br>";
echo "Επισκεφθείτε τον ιστότοπό μας: " . SITE_URL . "<br>";

// Σταθερές που ορίζονται από την PHP
echo "Η έκδοση της PHP είναι " . PHP_VERSION . "<br>";
echo "Το όριο μεγέθους αρχείου για μεταφορτώσεις είναι " . ini_get('upload_max_filesize') . "<br>";
?>

Διαφορές μεταξύ Μεταβλητών και Σταθερών:

  • Οι σταθερές δεν χρησιμοποιούν το σύμβολο $ μπροστά από το όνομά τους
  • Οι σταθερές μπορούν να προσπελαστούν οπουδήποτε στο σενάριο χωρίς να ενδιαφέρει το πεδίο (scope)
  • Οι σταθερές δεν μπορούν να επαναοριστούν ή να καταργηθούν αφού έχουν οριστεί
  • Τα ονόματα των σταθερών είναι συμβατικά με κεφαλαία γράμματα (αν και δεν είναι υποχρεωτικό)

5. Τελεστές

Οι τελεστές στην PHP είναι σύμβολα που χρησιμοποιούνται για την εκτέλεση λειτουργιών σε μεταβλητές και τιμές.

Αριθμητικοί Τελεστές

Χρησιμοποιούνται για την εκτέλεση βασικών μαθηματικών πράξεων:

<?php
$a = 10;
$b = 3;

// Βασικές αριθμητικές πράξεις
echo $a + $b;  // Πρόσθεση: 13
echo "<br>";
echo $a - $b;  // Αφαίρεση: 7
echo "<br>";
echo $a * $b;  // Πολλαπλασιασμός: 30
echo "<br>";
echo $a / $b;  // Διαίρεση: 3.3333...
echo "<br>";
echo $a % $b;  // Υπόλοιπο (Modulus): 1
echo "<br>";
echo $a ** $b; // Δύναμη (Exponentiation): 1000 (10^3)
echo "<br>";

// Αύξηση και μείωση
$c = 5;
echo $c++;    // Εμφανίζει 5, μετά αυξάνει το $c σε 6
echo "<br>";
echo $c;      // Εμφανίζει 6
echo "<br>";

$d = 5;
echo ++$d;    // Αυξάνει το $d σε 6, μετά εμφανίζει 6
echo "<br>";
?>

Τελεστές Εκχώρησης

Χρησιμοποιούνται για την ανάθεση τιμών σε μεταβλητές:

<?php
$a = 10;      // Βασική εκχώρηση
echo $a . "<br>";

$a += 5;      // Ισοδύναμο με: $a = $a + 5;
echo $a . "<br>";  // 15

$a -= 3;      // Ισοδύναμο με: $a = $a - 3;
echo $a . "<br>";  // 12

$a *= 2;      // Ισοδύναμο με: $a = $a * 2;
echo $a . "<br>";  // 24

$a /= 4;      // Ισοδύναμο με: $a = $a / 4;
echo $a . "<br>";  // 6

$a %= 4;      // Ισοδύναμο με: $a = $a % 4;
echo $a . "<br>";  // 2

$b = "Γεια ";
$b .= "σου!"; // Ισοδύναμο με: $b = $b . "σου!";
echo $b;      // "Γεια σου!"
?>

Τελεστές Σύγκρισης

Χρησιμοποιούνται για σύγκριση τιμών και επιστρέφουν boolean αποτελέσματα (true/false):

<?php
$a = 10;
$b = 5;
$c = "10";

// Βασικές συγκρίσεις
var_dump($a == $b);   // Ίσο: false
var_dump($a != $b);   // Διάφορο: true
var_dump($a > $b);    // Μεγαλύτερο από: true
var_dump($a < $b);    // Μικρότερο από: false
var_dump($a >= $b);   // Μεγαλύτερο ή ίσο: true
var_dump($a <= $b);   // Μικρότερο ή ίσο: false

// Συγκρίσεις με έλεγχο τύπου
var_dump($a == $c);   // Ίσο (χωρίς έλεγχο τύπου): true
var_dump($a === $c);  // Πανομοιότυπο (με έλεγχο τύπου): false
var_dump($a != $c);   // Διάφορο (χωρίς έλεγχο τύπου): false
var_dump($a !== $c);  // Όχι πανομοιότυπο (με έλεγχο τύπου): true

// Τελεστής Spaceship (από PHP 7)
// Επιστρέφει -1, 0 ή 1 αν το αριστερό είναι μικρότερο, ίσο ή μεγαλύτερο από το δεξί
echo $a <=> $b;  // 1 (το $a είναι μεγαλύτερο από το $b)
echo "<br>";
echo $b <=> $a;  // -1 (το $b είναι μικρότερο από το $a)
echo "<br>";
echo $a <=> $c;  // 0 (το $a είναι ίσο με το $c, αγνοώντας τον τύπο)
?>

Βέλτιστη Πρακτική: Όταν συγκρίνετε μεταβλητές, είναι συχνά προτιμότερο να χρησιμοποιείτε τους τελεστές αυστηρής ισότητας (=== και !==) που ελέγχουν τόσο την τιμή όσο και τον τύπο. Αυτό μπορεί να αποτρέψει δυσάρεστες εκπλήξεις λόγω της ιδιαιτερότητας της PHP στις μετατροπές τύπων.

Λογικοί Τελεστές

Χρησιμοποιούνται για τον συνδυασμό συνθηκών:

<?php
$a = true;
$b = false;

// Βασικοί λογικοί τελεστές
var_dump($a && $b);  // AND (Λογικό ΚΑΙ): false
var_dump($a || $b);  // OR (Λογικό Ή): true
var_dump(!$a);       // NOT (Λογικό ΟΧΙ): false

// Εναλλακτική σύνταξη
var_dump($a and $b); // AND: false
var_dump($a or $b);  // OR: true
var_dump($a xor $b); // XOR (αποκλειστικό Ή): true

// Παράδειγμα με συνθήκες
$age = 25;
$isStudent = true;

// Έλεγχος αν το άτομο είναι φοιτητής ΚΑΙ κάτω των 30 ετών
if ($isStudent && $age < 30) {
    echo "Νεαρός φοιτητής!";
}

// Έλεγχος αν το άτομο είναι φοιτητής Ή κάτω των 18 ετών
if ($isStudent || $age < 18) {
    echo "Ειδική κατηγορία!";
}
?>

Σημείωση για Προτεραιότητα: Οι τελεστές && και || έχουν υψηλότερη προτεραιότητα από τους and και or αντίστοιχα. Αυτό μπορεί να οδηγήσει σε απροσδόκητα αποτελέσματα, ειδικά κατά το συνδυασμό με τον τελεστή εκχώρησης. Για παράδειγμα:

$a = false || true;   // Το $a γίνεται true
$b = false or true;   // Το $b γίνεται false (η εκχώρηση έχει προτεραιότητα)

Τελεστές Συμβολοσειρών

Χρησιμοποιούνται για την ένωση (concatenation) συμβολοσειρών:

<?php
$string1 = "Γεια";
$string2 = "σου";

// Τελεστής concatenation (.)
echo $string1 . " " . $string2;  // "Γεια σου"
echo "<br>";

// Τελεστής εκχώρησης concatenation (.=)
$greeting = "Καλημέρα";
$greeting .= " και καλή εβδομάδα!";
echo $greeting;  // "Καλημέρα και καλή εβδομάδα!"
?>

Τελεστής Ternary

Ένας συνοπτικός τρόπος για να γράψετε απλές δομές εντολών if-else:

<?php
$age = 20;

// Κλασική δομή if-else
if ($age >= 18) {
    $status = "Ενήλικας";
} else {
    $status = "Ανήλικος";
}
echo $status . "<br>";  // "Ενήλικας"

// Ισοδύναμη χρήση του τελεστή ternary
$status = ($age >= 18) ? "Ενήλικας" : "Ανήλικος";
echo $status . "<br>";  // "Ενήλικας"

// Συντομογραφία ternary (από PHP 5.3)
$username = isset($user) ? $user : "Επισκέπτης";
// Μπορεί επίσης να γραφτεί ως:
$username = $user ?? "Επισκέπτης";  // Τελεστής ?? (Null coalescing, από PHP 7)
echo $username;  // "Επισκέπτης" (αν το $user δεν έχει οριστεί)
?>

Επεξήγηση:

  • Η συνθήκη μέσα στην if πρέπει να αξιολογείται σε true για να εκτελεστεί ο κώδικας μέσα στο μπλοκ της.
  • Όταν μια συνθήκη if είναι false, ελέγχεται η επόμενη συνθήκη else if (αν υπάρχει).
  • Αν καμία συνθήκη δεν είναι true, εκτελείται ο κώδικας στο μπλοκ else (αν υπάρχει).

Επεξήγηση:

  • Η δομή switch αξιολογεί μια έκφραση και στη συνέχεια εκτελεί τον αντίστοιχο κώδικα με βάση την τιμή της έκφρασης.
  • Η εντολή break τερματίζει την εκτέλεση της δομής switch. Χωρίς αυτήν, η εκτέλεση θα συνεχίσει στην επόμενη περίπτωση (fall-through).
  • Η περίπτωση default εκτελείται όταν καμία άλλη περίπτωση δεν ταιριάζει.
  • Το fall-through μπορεί να είναι χρήσιμο όταν θέλετε ο ίδιος κώδικας να εκτελεστεί για πολλαπλές περιπτώσεις, όπως στο παράδειγμα των δικαιωμάτων χρήστη.

Βρόχος while

Ο βρόχος while εκτελεί ένα μπλοκ κώδικα όσο μια συνθήκη είναι αληθής:

<?php
// Βασικός βρόχος while
$i = 1;
while ($i <= 5) {
    echo "Αριθμός: $i" . "<br>";
    $i++;
}
/* Εμφανίζει:
Αριθμός: 1
Αριθμός: 2
Αριθμός: 3
Αριθμός: 4
Αριθμός: 5
*/

// Παράδειγμα με χρήση συνάρτησης
$colors = array("κόκκινο", "πράσινο", "μπλε", "κίτρινο");
$i = 0;
while ($i < count($colors)) {
    echo "Χρώμα: " . $colors[$i] . "<br>";
    $i++;
}

// Εναλλακτική σύνταξη για templates
$j = 1;
while ($j <= 3):
    echo "Στοιχείο $j" . "<br>";
    $j++;
endwhile;
?>

Επεξήγηση:

  • Η συνθήκη αξιολογείται πριν από κάθε επανάληψη του βρόχου.
  • Αν η συνθήκη είναι false από την αρχή, το μπλοκ κώδικα δεν θα εκτελεστεί ποτέ.
  • Είναι σημαντικό να διασφαλίσετε ότι η συνθήκη κάποια στιγμή θα γίνει false, διαφορετικά θα δημιουργηθεί ένας ατέρμονος βρόχος.

Βρόχος do...while

Παρόμοιος με τον βρόχο while, αλλά αξιολογεί τη συνθήκη μετά την εκτέλεση του μπλοκ κώδικα, διασφαλίζοντας ότι ο βρόχος θα εκτελεστεί τουλάχιστον μία φορά:

<?php
$i = 1;
do {
    echo "Αριθμός: $i" . "<br>";
    $i++;
} while ($i <= 5);
/* Εμφανίζει:
Αριθμός: 1
Αριθμός: 2
Αριθμός: 3
Αριθμός: 4
Αριθμός: 5
*/

// Ακόμα και αν η συνθήκη είναι αρχικά false, ο βρόχος εκτελείται μία φορά
$j = 10;
do {
    echo "Αυτό θα εκτελεστεί μία φορά, παρόλο που η συνθήκη είναι false" . "<br>";
} while ($j < 5);
?>

Η κύρια διαφορά μεταξύ while και do...while:

  • Στον βρόχο while, η συνθήκη ελέγχεται πριν την εκτέλεση του κώδικα.
  • Στον βρόχο do...while, ο κώδικας εκτελείται πρώτα και μετά ελέγχεται η συνθήκη.
  • Αυτό σημαίνει ότι ο βρόχος do...while θα εκτελεστεί τουλάχιστον μία φορά, ακόμα και αν η συνθήκη είναι αρχικά false.

    Εναλλακτικές Ετικέτες PHP

    Εκτός από την τυπική ετικέτα <?php ?>, υπάρχουν και άλλες ετικέτες που μπορούν να χρησιμοποιηθούν:

    <?php echo "Τυπική ετικέτα"; ?>

<? echo "Σύντομη ετικέτα (συνήθως απενεργοποιημένη από προεπιλογή)"; ?>

<?= "Συντόμευση της echo (ισοδύναμη με <?php echo)"; ?>

    Επεξήγηση:

    • Η σύντομη ετικέτα <? ?> μπορεί να μην είναι διαθέσιμη σε όλες τις εγκαταστάσεις PHP (εξαρτάται από τη ρύθμιση short_open_tag)
    • Η ετικέτα <?= ?> είναι μια συντόμευση για την <?php echo ?> και είναι πάντα διαθέσιμη από την PHP 5.4 και μετά
    • Για τη μέγιστη συμβατότητα, συνιστάται η χρήση της τυπικής ετικέτας <?php ?>

    Εκτέλεση Κώδικα PHP

    Είναι σημαντικό να καταλάβουμε ότι ο κώδικας PHP εκτελείται στον διακομιστή, και το αποτέλεσμα αποστέλλεται στον περιηγητή ως HTML:

    1. Ο χρήστης ζητά μια σελίδα PHP από τον διακομιστή
    2. Ο διακομιστής ιστού αναγνωρίζει το αρχείο ως PHP και το περνά στον επεξεργαστή PHP
    3. Ο επεξεργαστής PHP εκτελεί όλο τον κώδικα που βρίσκεται μεταξύ των ετικετών <?php και ?>
    4. Ο PHP επεξεργαστής επιστρέφει το αποτέλεσμα στον διακομιστή ιστού
    5. Ο διακομιστής ιστού στέλνει το αποτέλεσμα (HTML, CSS, JavaScript) στον περιηγητή του χρήστη
    6. Ο περιηγητής παρουσιάζει τη σελίδα στον χρήστη

    Αυτό σημαίνει ότι ο χρήστης δεν βλέπει ποτέ τον πραγματικό κώδικα PHP, αλλά μόνο το αποτέλεσμα που παράγεται από αυτόν.

    Case Sensitivity

    Στην PHP, η ευαισθησία στα κεφαλαία/πεζά (case sensitivity) ισχύει ως εξής:

    • Case-Insensitive: Ονόματα συναρτήσεων, ονόματα κλάσεων, λέξεις-κλειδιά της PHP (if, else, while, κλπ.)
    • Case-Sensitive: Ονόματα μεταβλητών, ονόματα σταθερών (εκτός από τις προκαθορισμένες), ονόματα labels, ονόματα κλειδιών πινάκων
    <?php
$age = 25;        // Μεταβλητή με πεζά
$AGE = 30;        // Διαφορετική μεταβλητή με κεφαλαία

echo $age;        // Εμφανίζει 25
echo $AGE;        // Εμφανίζει 30

ECHO "Κείμενο";   // Λειτουργεί (οι συναρτήσεις δεν είναι case-sensitive)
echo "Κείμενο";   // Λειτουργεί
EcHo "Κείμενο";   // Λειτουργεί

if ($age < 30) {  // Το "if" λειτουργεί με οποιονδήποτε συνδυασμό κεφαλαίων/πεζών
    echo "Νέος!";
}
?>

    Βέλτιστη Πρακτική: Παρόλο που κάποια στοιχεία της PHP δεν είναι ευαίσθητα στα κεφαλαία/πεζά, συνιστάται να ακολουθείτε μια συνεπή σύμβαση ονοματοδοσίας για καλύτερη αναγνωσιμότητα και συντήρηση του κώδικα.

    • Πλεονεκτήματα της PHP

    • Ευκολία εκμάθησης: Η σύνταξη της PHP είναι σχετικά απλή και εύκολη στην εκμάθηση, ειδικά για όσους έχουν ήδη εμπειρία με γλώσσες όπως η C, Java ή JavaScript.
    • Πολυπλατφορμική: Λειτουργεί σε διάφορα λειτουργικά συστήματα (Windows, Linux, macOS, κλπ.) και με διάφορους διακομιστές ιστού.
    • Ευρεία κοινότητα: Έχει μια τεράστια κοινότητα προγραμματιστών και πληθώρα βιβλιοθηκών, πλαισίων (frameworks) και πόρων.
    • Ενσωμάτωση με βάσεις δεδομένων: Παρέχει εξαιρετική υποστήριξη για πολλά συστήματα διαχείρισης βάσεων δεδομένων.
    • Ευελιξία: Μπορεί να χρησιμοποιηθεί για απλές ιστοσελίδες μέχρι πολύπλοκες εφαρμογές ιστού.

    7. Συναρτήσεις

    Οι συναρτήσεις είναι επαναχρησιμοποιήσιμα τμήματα κώδικα που εκτελούν συγκεκριμένες εργασίες. Η PHP διαθέτει πολλές ενσωματωμένες συναρτήσεις, αλλά επιτρέπει και τον ορισμό προσαρμοσμένων συναρτήσεων.

    Ορισμός και Κλήση Συναρτήσεων

    Μια συνάρτηση στην PHP ορίζεται χρησιμοποιώντας τη λέξη-κλειδί function:

    <?php
// Ορισμός απλής συνάρτησης
function sayHello() {
    echo "Γεια σας!";
}

// Κλήση της συνάρτησης
sayHello(); // Εμφανίζει: Γεια σας!

// Συνάρτηση με παραμέτρους
function greet($name) {
    echo "Γεια σου, $name!";
}

greet("Μαρία"); // Εμφανίζει: Γεια σου, Μαρία!

// Συνάρτηση με πολλαπλές παραμέτρους
function addNumbers($a, $b) {
    $sum = $a + $b;
    echo "Το άθροισμα του $a και $b είναι: $sum";
}

addNumbers(5, 3); // Εμφανίζει: Το άθροισμα του 5 και 3 είναι: 8
?>

    Επιστροφή Τιμών

    Οι συναρτήσεις μπορούν να επιστρέψουν τιμές χρησιμοποιώντας τη δήλωση return:

    <?php
// Συνάρτηση που επιστρέφει μια τιμή
function square($number) {
    return $number * $number;
}

// Αποθήκευση της επιστρεφόμενης τιμής σε μεταβλητή
$result = square(4);
echo "Το τετράγωνο του 4 είναι: $result"; // Εμφανίζει: Το τετράγωνο του 4 είναι: 16

// Χρήση της επιστρεφόμενης τιμής απευθείας σε έκφραση
echo "Το τετράγωνο του 5 είναι: " . square(5); // Εμφανίζει: Το τετράγωνο του 5 είναι: 25

// Συνάρτηση που επιστρέφει πολλαπλές τιμές (ως πίνακα)
function getMinMax($numbers) {
    return [
        'min' => min($numbers),
        'max' => max($numbers)
    ];
}

$stats = getMinMax([3, 1, 7, 5]);
echo "Ελάχιστο: " . $stats['min'] . ", Μέγιστο: " . $stats['max'];
// Εμφανίζει: Ελάχιστο: 1, Μέγιστο: 7
?>

    Σημείωση: Μια συνάρτηση μπορεί να έχει πολλές δηλώσεις return, αλλά η εκτέλεση της συνάρτησης θα σταματήσει μόλις συναντήσει μια δήλωση return. Αν δεν συναντήσει καμία, η συνάρτηση επιστρέφει null σιωπηρά.

    Προεπιλεγμένες Τιμές Παραμέτρων

    Μπορείτε να ορίσετε προεπιλεγμένες τιμές για τις παραμέτρους, οι οποίες θα χρησιμοποιηθούν αν δεν παρέχονται τιμές κατά την κλήση της συνάρτησης:

    <?php
// Συνάρτηση με προεπιλεγμένες τιμές παραμέτρων
function makeGreeting($name, $greeting = "Γεια σου") {
    return "$greeting, $name!";
}

echo makeGreeting("Γιάννη"); // Εμφανίζει: Γεια σου, Γιάννη!
echo "<br>";
echo makeGreeting("Μαρία", "Καλημέρα"); // Εμφανίζει: Καλημέρα, Μαρία!

// Συνάρτηση με πολλαπλές προεπιλεγμένες παραμέτρους
function formatName($first, $last, $title = "", $suffix = "") {
    $formattedName = "";
    
    if (!empty($title)) {
        $formattedName .= "$title ";
    }
    
    $formattedName .= "$first $last";
    
    if (!empty($suffix)) {
        $formattedName .= ", $suffix";
    }
    
    return $formattedName;
}

echo "<br>";
echo formatName("Γιώργος", "Παπαδόπουλος"); // Εμφανίζει: Γιώργος Παπαδόπουλος
echo "<br>";
echo formatName("Μαρία", "Νικολάου", "Δρ."); // Εμφανίζει: Δρ. Μαρία Νικολάου
echo "<br>";
echo formatName("Ιωάννης", "Γεωργίου", "Καθ.", "PhD"); // Εμφανίζει: Καθ. Ιωάννης Γεωργίου, PhD
?>

    Σημαντικό: Οι παράμετροι με προεπιλεγμένες τιμές πρέπει να βρίσκονται στο τέλος της λίστας παραμέτρων. Δεν μπορείτε να έχετε μια προεπιλεγμένη παράμετρο πριν από μια μη προεπιλεγμένη.

    Μεταβλητός Αριθμός Παραμέτρων

    Η PHP επιτρέπει τη δημιουργία συναρτήσεων που δέχονται μεταβλητό αριθμό παραμέτρων χρησιμοποιώντας τον τελεστή ... (από την PHP 5.6 και μετά):

    <?php
// Συνάρτηση με μεταβλητό αριθμό παραμέτρων
function sum(...$numbers) {
    $total = 0;
    foreach ($numbers as $num) {
        $total += $num;
    }
    return $total;
}

echo sum(1, 2, 3, 4); // Εμφανίζει: 10
echo "<br>";
echo sum(5, 10, 15); // Εμφανίζει: 30

// Συνδυασμός κανονικών και μεταβλητών παραμέτρων
function teamGreeting($greeting, ...$names) {
    $message = "$greeting ";
    $lastIndex = count($names) - 1;
    
    for ($i = 0; $i < count($names); $i++) {
        if ($i === $lastIndex && $i !== 0) {
            $message .= "και ";
        } elseif ($i > 0) {
            $message .= ", ";
        }
        
        $message .= $names[$i];
    }
    
    return $message . "!";
}

echo "<br>";
echo teamGreeting("Καλημέρα", "Γιάννη"); // Εμφανίζει: Καλημέρα Γιάννη!
echo "<br>";
echo teamGreeting("Γεια σας", "Μαρία", "Νίκο", "Ελένη"); // Εμφανίζει: Γεια σας Μαρία, Νίκο και Ελένη!
?>

    Παλαιότερος τρόπος (πριν την PHP 5.6): Για παλαιότερες εκδόσεις της PHP, οι συναρτήσεις func_num_args(), func_get_arg() και func_get_args() χρησιμοποιούνταν για να χειριστούν μεταβλητό αριθμό παραμέτρων.

    Εμβέλεια Μεταβλητών (Variable Scope)

    Η εμβέλεια αναφέρεται στις περιοχές του κώδικα όπου μια μεταβλητή είναι προσβάσιμη. Στην PHP, οι μεταβλητές έχουν συνήθως τοπική εμβέλεια, εκτός αν δηλωθούν διαφορετικά:

    <?php
// Καθολική μεταβλητή
$globalVar = "Είμαι καθολική";

function testScope() {
    // Τοπική μεταβλητή
    $localVar = "Είμαι τοπική";
    
    // Η τοπική μεταβλητή είναι προσβάσιμη εδώ
    echo "Μέσα στη συνάρτηση: $localVar" . "<br>";
    
    // Προσπαθώντας να προσπελάσουμε την καθολική μεταβλητή
    // (δεν θα λειτουργήσει χωρίς τη λέξη-κλειδί global)
    echo "Προσπαθώντας να προσπελάσουμε την καθολική: $globalVar" . "<br>";
    
    // Χρήση της λέξης-κλειδί global για πρόσβαση στην καθολική μεταβλητή
    global $globalVar;
    echo "Μετά τη χρήση του global: $globalVar" . "<br>";
    
    // Εναλλακτικά, χρήση του πίνακα $GLOBALS
    echo "Χρησιμοποιώντας \$GLOBALS: " . $GLOBALS['globalVar'] . "<br>";
}

testScope();

// Η τοπική μεταβλητή δεν είναι προσβάσιμη εδώ
// Το παρακάτω θα προκαλέσει σφάλμα ή θα εμφανίσει κενό
echo "Έξω από τη συνάρτηση, προσπαθώντας να προσπελάσουμε την τοπική: $localVar" . "<br>";

// Η καθολική μεταβλητή είναι προσβάσιμη εδώ
echo "Έξω από τη συνάρτηση, προσπελαύνοντας την καθολική: $globalVar" . "<br>";
?>

    Βέλτιστη Πρακτική: Η υπερβολική χρήση καθολικών μεταβλητών δεν συνιστάται, καθώς μπορεί να οδηγήσει σε κώδικα που είναι δύσκολο να διαβαστεί, να διατηρηθεί και να αποσφαλματωθεί. Είναι προτιμότερο να περνάτε δεδομένα ως παραμέτρους και να επιστρέφετε τιμές από τις συναρτήσεις.

    <?php
function countCalls() {

*/

/* Εμφανίζει:

    Ανώνυμες Συναρτήσεις (Lambdas)

    Οι ανώνυμες συναρτήσεις, γνωστές και ως λάμδα συναρτήσεις, είναι συναρτήσεις χωρίς όνομα που μπορούν να εκχωρηθούν σε μεταβλητές ή να περαστούν ως παράμετροι:

    <?php
// Ορισμός ανώνυμης συνάρτησης και εκχώρησή της σε μια μεταβλητή
$greet = function($name) {
    return "Γεια σου, $name!";
};

// Κλήση της ανώνυμης συνάρτησης
echo $greet("Μαρία"); // Εμφανίζει: Γεια σου, Μαρία!
echo "<br>";

// Χρήση ανώνυμης συνάρτησης ως παράμετρο
function executeCallback($callback, $value) {
    return $callback($value);
}

$double = function($x) {
    return $x * 2;
};

echo executeCallback($double, 5); // Εμφανίζει: 10
echo "<br>";

// Απευθείας πέρασμα ανώνυμης συνάρτησης ως παράμετρο
echo executeCallback(function($x) { return $x * $x; }, 4); // Εμφανίζει: 16
?>

    Χρήση των arrow functions (από PHP 7.4)

    Οι arrow functions είναι μια σύντομη σύνταξη για τις ανώνυμες συναρτήσεις, ιδανική για απλές λειτουργίες:

    <?php
// Κλασική ανώνυμη συνάρτηση
$multiply = function($a, $b) {
    return $a * $b;
};

// Ισοδύναμη arrow function
$multiply = fn($a, $b) => $a * $b;

echo $multiply(5, 3); // Εμφανίζει: 15
echo "<br>";

// Παράδειγμα με array_map
$numbers = [1, 2, 3, 4, 5];

// Χρησιμοποιώντας arrow function με array_map
$squared = array_map(fn($n) => $n * $n, $numbers);

// Εμφάνιση του αποτελέσματος
echo "Οι τετραγωνικοί αριθμοί είναι: ";
echo implode(', ', $squared); // Εμφανίζει: Οι τετραγωνικοί αριθμοί είναι: 1, 4, 9, 16, 25
?>

    Πλεονεκτήματα των arrow functions:

    • Πιο σύντομη και καθαρή σύνταξη
    • Αυτόματη πρόσβαση στις μεταβλητές του περιβάλλοντος πεδίου (δεν χρειάζεται η χρήση του use)
    • Περιορίζονται σε μια μόνο έκφραση επιστροφής, που ενθαρρύνει απλές, σαφείς συναρτήσεις

    Αναδρομικές Συναρτήσεις

    Μια αναδρομική συνάρτηση είναι μια συνάρτηση που καλεί τον εαυτό της:

    <?php
// Κλασικό παράδειγμα αναδρομής: υπολογισμός παραγοντικού
function factorial($n) {
    // Βασική περίπτωση
    if ($n <= 1) {
        return 1;
    }
    
    // Αναδρομική περίπτωση
    return $n * factorial($n - 1);
}

echo "Το παραγοντικό του 5 είναι: " . factorial(5); // Εμφανίζει: Το παραγοντικό του 5 είναι: 120
echo "<br>";

// Παράδειγμα αναδρομής: αριθμοί Fibonacci
function fibonacci($n) {
    // Βασικές περιπτώσεις
    if ($n <= 0) {
        return 0;
    }
    if ($n == 1) {
        return 1;
    }
    
    // Αναδρομική περίπτωση
    return fibonacci($n - 1) + fibonacci($n - 2);
}

echo "Ο 10ος αριθμός Fibonacci είναι: " . fibonacci(10); // Εμφανίζει: Ο 10ος αριθμός Fibonacci είναι: 55
?>

    Προσοχή: Οι αναδρομικές συναρτήσεις πρέπει να χρησιμοποιούνται προσεκτικά, καθώς μπορεί να καταναλώσουν σημαντική μνήμη και χρόνο επεξεργασίας για μεγάλες τιμές εισόδου. Πάντα να περιλαμβάνετε μια βασική περίπτωση για να σταματήσει η αναδρομή.

    • Χειρισμός Συμβολοσειρών: strlen(), strpos(), str_replace(), substr(), trim(), κ.λπ.
    • Συναρτήσεις Πινάκων: count(), array_push(), array_pop(), array_merge(), sort(), κ.λπ.
    • Μαθηματικές Συναρτήσεις: abs(), round(), floor(), ceil(), max(), min(), rand(), κ.λπ.
    • Συναρτήσεις Ημερομηνίας και Ώρας: date(), time(), strtotime(), mktime(), κ.λπ.
    • Συναρτήσεις Αρχείων: file_get_contents(), file_put_contents(), fopen(), fread(), fwrite(), fclose(), κ.λπ.
    • Συναρτήσεις Χειρισμού URL: parse_url(), urlencode(), urldecode(), κ.λπ.

    • Συναρτήσεις Πληροφοριών Μεταβλητών: isset(), empty(), is_array(), is_string(), gettype(), κ.λπ.

Καθώς εξελίσσεται η γλώσσα, νέα χαρακτηριστικά όπως οι arrow functions προστίθενται για να κάνουν τον κώδικα πιο συνοπτικό και εκφραστικό.     9. Υπερκαθολικές Μεταβλητές

    Οι υπερκαθολικές μεταβλητές (superglobals) είναι προκαθορισμένοι πίνακες στην PHP που είναι πάντα διαθέσιμοι σε όλα τα πεδία εφαρμογής. Αυτοί οι πίνακες περιέχουν πληροφορίες από διάφορες πηγές, όπως δεδομένα φορμών, cookies, μεταβλητές συνεδρίας, και περιβαλλοντικές μεταβλητές.

    Διαθέσιμες Υπερκαθολικές Μεταβλητές

    Η PHP παρέχει τις ακόλουθες υπερκαθολικές μεταβλητές:

    Υπερκαθολική Περιγραφή
    $_GET Περιέχει τις μεταβλητές που περνούν στο script μέσω των παραμέτρων URL (query string)
    $_POST Περιέχει τις μεταβλητές που αποστέλλονται στο script μέσω της μεθόδου HTTP POST
    $_REQUEST Περιέχει τα περιεχόμενα των πινάκων $_GET, $_POST και $_COOKIE
    $_COOKIE Περιέχει τις μεταβλητές που περνούν στο script μέσω HTTP cookies
    $_SESSION Περιέχει τις μεταβλητές συνεδρίας που είναι διαθέσιμες στο τρέχον script
    $_FILES Περιέχει πληροφορίες για αρχεία που ανέβηκαν μέσω HTTP POST
    $_SERVER Περιέχει πληροφορίες όπως επικεφαλίδες, διαδρομές και τοποθεσίες scripts
    $_ENV Περιέχει μεταβλητές που παρέχονται στο script από το περιβάλλον του server
    $GLOBALS Αναφέρεται σε όλες τις μεταβλητές που είναι διαθέσιμες στο παγκόσμιο πεδίο του script

    $_GET

    Η μεταβλητή $_GET χρησιμοποιείται για τη συλλογή δεδομένων που αποστέλλονται στο URL μέσω των παραμέτρων query string.

    <?php
// Για ένα URL όπως: example.php?name=Μαρία&age=25

// Πρόσβαση σε παραμέτρους GET
if (isset($_GET['name']) && isset($_GET['age'])) {
    $name = $_GET['name'];
    $age = $_GET['age'];
    
    echo "Όνομα: $name<br>";
    echo "Ηλικία: $age";
} else {
    echo "Παρακαλώ παρέχετε όνομα και ηλικία στο URL.";
}

// Μπορείτε να δημιουργήσετε συνδέσμους με παραμέτρους GET
echo '<a href="profile.php?id=123&type=user">Προβολή Προφίλ</a>';
?>

    Ασφάλεια: Τα δεδομένα από την $_GET είναι ορατά στη γραμμή URL και δεν πρέπει να χρησιμοποιούνται για ευαίσθητες πληροφορίες. Επίσης, πάντα να επικυρώνετε και να καθαρίζετε τα δεδομένα από τον χρήστη για να αποτρέψετε επιθέσεις όπως το SQL Injection και το Cross-Site Scripting (XSS).

    $_REQUEST

    Η μεταβλητή $_REQUEST περιέχει τα δεδομένα από $_GET, $_POST και $_COOKIE. Είναι λιγότερο συγκεκριμένη, αλλά μπορεί να είναι βολική όταν δεν ενδιαφέρεστε για την πηγή των δεδομένων.

// 1. Ελέγξτε αν η μεταβλητή υπάρχει
if (isset($_POST['username'])) {

    $username = trim($_POST['username']);

    $username = strip_tags($username);
    
    // 4. Μετατρέψτε ειδικούς χαρακτήρες σε HTML entities
    $username = htmlspecialchars($username, ENT_QUOTES, 'UTF-8');
    
    // 5. Επικυρώστε το μήκος και το μοτίβο αν χρειάζεται
    if (strlen($username) < 3 || strlen($username) > 20) {
        echo "Το όνομα χρήστη πρέπει να έχει μεταξύ 3 και 20 χαρακτήρες.";
        exit;
    }
    
    // 6. Χρησιμοποιήστε προετοιμασμένες δηλώσεις για ερωτήματα SQL
    // Παράδειγμα με PDO:
    $stmt = $pdo->prepare("SELECT * FROM users WHERE username = ?");
    $stmt->execute([$username]);
}
?>

    Σημαντικό: Ποτέ μην εμπιστεύεστε δεδομένα που προέρχονται από τον χρήστη. Πάντα να επικυρώνετε, να φιλτράρετε και να καθαρίζετε τα δεδομένα πριν τα χρησιμοποιήσετε, ειδικά όταν χρησιμοποιούνται σε ερωτήματα βάσης δεδομένων, δημιουργία αρχείων ή εκτέλεση κώδικα.

    10. Χειρισμός Φορμών

    Ασφάλεια Φορμών

    Η ασφάλεια είναι κρίσιμη κατά τον χειρισμό δεδομένων φόρμας. Ακολουθούν μερικές βασικές τεχνικές για την προστασία των φορμών σας:

    1. Πρόληψη επιθέσεων Cross-Site Scripting (XSS)

    <?php
// ΛΑΘΟΣ - Επικίνδυνο
echo "Γεια σου, " . $_POST['name'];

// ΣΩΣΤΟ - Ασφαλές
echo "Γεια σου, " . htmlspecialchars($_POST['name'], ENT_QUOTES, 'UTF-8');
?>

    2. Πρόληψη επιθέσεων Cross-Site Request Forgery (CSRF)

    <?php
// Ξεκινήστε συνεδρία
session_start();

// Δημιουργία ενός τυχαίου token και αποθήκευσή του στη συνεδρία
if (!isset($_SESSION['csrf_token'])) {
    $_SESSION['csrf_token'] = bin2hex(random_bytes(32));
}

// Στη φόρμα HTML, προσθέστε ένα κρυφό πεδίο με το token
?>

<form method="post" action="process.php">
    <!-- Άλλα πεδία φόρμας -->
    <input type="hidden" name="csrf_token" value="<?php echo $_SESSION['csrf_token']; ?>">
    <input type="submit" value="Υποβολή">
</form>

<?php
// Στο process.php, επαληθεύστε το token
if ($_SERVER["REQUEST_METHOD"] == "POST") {
    if (!isset($_POST['csrf_token']) || $_POST['csrf_token'] !== $_SESSION['csrf_token']) {
        die('Η επαλήθευση CSRF απέτυχε');
    }
    
    // Συνεχίστε με την επεξεργασία της φόρμας
}
?>

    3. Πρόληψη επιθέσεων SQL Injection

    <?php
// ΛΑΘΟΣ - Επικίνδυνο
$username = $_POST['username'];
$query = "SELECT * FROM users WHERE username = '$username'";

// ΣΩΣΤΟ - Χρήση προετοιμασμένων δηλώσεων με PDO
$username = $_POST['username'];
$stmt = $pdo->prepare("SELECT * FROM users WHERE username = ?");
$stmt->execute([$username]);
$user = $stmt->fetch();
?>

    Βέλτιστες Πρακτικές για Χειρισμό Φορμών

    1. Επικύρωση σε δύο επίπεδα: Εφαρμόστε επικύρωση τόσο στον client (JavaScript) όσο και στον server (PHP).
    2. Καθαρισμός δεδομένων: Πάντα να καθαρίζετε και να φιλτράρετε τα δεδομένα εισόδου πριν τα χρησιμοποιήσετε.
    3. Εμφάνιση σφαλμάτων: Παρέχετε σαφή και συγκεκριμένα μηνύματα σφαλμάτων για τον χρήστη.
    4. Διατήρηση τιμών: Διατηρήστε τις τιμές της φόρμας μετά από μια αποτυχημένη υποβολή.
    5. Περιορισμός πεδίων: Χρησιμοποιήστε τα χαρακτηριστικά maxlength, min, max για να περιορίσετε την είσοδο δεδομένων.
    6. Προσβασιμότητα: Χρησιμοποιήστε ετικέτες <label> και συσχετίστε τες με τα αντίστοιχα πεδία εισόδου.
    7. Χρήση προεπιλεγμένων τιμών: Όπου είναι λογικό, παρέχετε προεπιλεγμένες τιμές για τα πεδία.
    8. Ομαδοποίηση σχετικών πεδίων: Χρησιμοποιήστε <fieldset> και <legend> για την ομαδοποίηση σχετικών πεδίων.
    9. Ασφαλείς HTTP μέθοδοι: Χρησιμοποιήστε POST για ευαίσθητα δεδομένα και για φόρμες που αλλάζουν την κατάσταση του συστήματος.
    10. Έλεγχοι στην πλευρά του διακομιστή: Μην βασίζεστε μόνο στην επικύρωση client-side.

    Αντιμετώπιση Προβλημάτων Κωδικοποίησης Χαρακτήρων

    Όταν εργάζεστε με ελληνικούς χαρακτήρες, είναι σημαντικό να διασφαλίσετε τη σωστή κωδικοποίηση για να αποφύγετε προβλήματα με ειδικούς χαρακτήρες:

    <!DOCTYPE html>
<html>
<head>
    <meta charset="UTF-8">
    <title>Φόρμα με Ελληνικούς Χαρακτήρες</title>
</head>
<body>
    <form method="post" action="<?php echo htmlspecialchars($_SERVER["PHP_SELF"]); ?>">
        <label for="greek_text">Κείμενο στα Ελληνικά:</label>
        <input type="text" id="greek_text" name="greek_text"><br><br>
        <input type="submit" value="Υποβολή">
    </form>
    
    <?php
    if ($_SERVER["REQUEST_METHOD"] == "POST") {
        // Βεβαιωθείτε ότι ο διακομιστής χρησιμοποιεί UTF-8
        header('Content-Type: text/html; charset=utf-8');
        
        $greekText = $_POST['greek_text'] ?? '';
        
        // Εάν είναι απαραίτητο, μετατρέψτε την κωδικοποίηση
        if (!mb_check_encoding($greekText, 'UTF-8')) {
            $greekText = mb_convert_encoding($greekText, 'UTF-8');
        }
        
        echo "Εισάγατε το εξής κείμενο: " . htmlspecialchars($greekText, ENT_QUOTES, 'UTF-8');
    }
    ?>
</body>
</html>

    Συμβουλές για τη διαχείριση ελληνικών χαρακτήρων:

    • Χρησιμοποιήστε πάντα κωδικοποίηση UTF-8 στις σελίδες HTML σας.
    • Ορίστε το charset UTF-8 στην επικεφαλίδα HTTP.
    • Βεβαιωθείτε ότι η βάση δεδομένων σας χρησιμοποιεί κωδικοποίηση UTF-8.
    • Χρησιμοποιήστε τις συναρτήσεις `mb_*` (multibyte string functions) για χειρισμό συμβολοσειρών.

    Συμπέρασμα

    Ο αποτελεσματικός χειρισμός φορμών είναι θεμελιώδης για τη δημιουργία διαδραστικών εφαρμογών ιστού. Η κατανόηση της συλλογής, επικύρωσης και ασφαλούς επεξεργασίας των δεδομένων φόρμας είναι απαραίτητη για κάθε προγραμματιστή PHP. Με την εφαρμογή των βέλτιστων πρακτικών που περιγράφονται σε αυτή την ενότητα, μπορείτε να αναπτύξετε φόρμες που είναι φιλικές προς το χρήστη, ασφαλείς και αξιόπιστες.

    Οι φόρμες λειτουργούν ως γέφυρα μεταξύ του χρήστη και της εφαρμογής σας, επιτρέποντας την αλληλεπίδραση και την ανταλλαγή δεδομένων. Ο προσεκτικός σχεδιασμός και η υλοποίηση των φορμών συμβάλλουν σημαντικά στη συνολική εμπειρία χρήστη της εφαρμογής σας.
